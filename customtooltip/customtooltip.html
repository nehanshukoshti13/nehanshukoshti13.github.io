Custom/report page tooltip in Power Bi

Report page or custom tooltips in Power Bi help us display data in depth and with clearity in our tooltips allowing us to display more data in a visually appealing and interactive way.

Contents of this article.

  • - Creating and using a report page tooltip
  • - Understanding the context and working of the report page tooltip


Creating and using a custom tooltip:
Crete a new page in Power BI desktop and hide the page from the page navigation at the bottom. In the created page go to the visualizations page and in "Page Information" turn on the tooltip toggle

Turn new page to tooltip page

In the visualization pane now in the "Page Size" section select type as Tooltip page.

Select pagesize type Tooltip

After completing the configurations, add the visuals you need to display in the tooltip on the page. Configure the tooltip as required. You can add dates or status or as in this case we have added count of employees per gender as a barchart in the tooltip.

VISUAL ON TOOLTIP PAGE

Below is the visual displaying count of data by salary grade/position with default tooltip. In the visualizations pane click on paint brush (format) visual pane. In default tooltip is the data value present in the visual, and we can add measures to display custom data, other way to do so is by adding report page tooltip.

default tooltip on visual

To add a tooltip select the visual you want to add a tooltip on. In the side bar select the format visualtions bar go to tooltip and add Report Page in the "Type" option and select the created tool tip page in the "Page" option.

How to add custom tooltip

Once you add the tooltip you could see the custom tooltip page with data filtered in context of visual you hover on.

Custom tooltip preview

There is one limmitation for the custom report page tooltip and that is the unresponsive nature of the tooltip. You can not interact with data inside the tooltip which may cause issue if you choose to use a table visual inside the tooltip with data not fitting inside screen and require scrolling as that is not possible in the tooltip.
Context of filter: To use the tooltip on a visual the context is required to filter data correctly in the tooltip. For the same the data displayed in the visual and tooltip need to have a direct relation. In our case the employee gender and position are connected through Employee, so the count of employee is filtered in context of pay grade/position in the main visual first, and then the filtered data is passed to tooltip visual and from that data the count of male and female employee is displayed in the tooltip. So to use the tooltip report page make sure to have a relation to establish the filter context.
